## 1. V4L2 框架概述

Video4Linux2 (V4L2) 是 Linux 内核中用于视频采集设备的标准框架，架构如下：

```
用户态应用
    │  open/ioctl/read/mmap
    ▼
/dev/videoN
    │
V4L2 核心层 (drivers/media/v4l2-core/)
    │  v4l2_device / v4l2_subdev / videobuf2
    ▼
具体设备驱动 (e.g., drivers/media/i2c/imx415.c)
    │
硬件抽象层 (MIPI CSI 控制器 / ISP)
    │
物理硬件 (IMX415 MIPI Camera)
```

## 2. 核心数据结构

### 2.1 v4l2_device — 顶层设备容器

```c
struct v4l2_device {
    struct device *dev;
    struct media_device *mdev;
    struct list_head subdevs;       // 关联的子设备链表
    char name[V4L2_DEVICE_NAME_SIZE];
    /* ... */
};
```

**要点**：每个 V4L2 驱动通过 `v4l2_device_register()` 注册。

### 2.2 v4l2_subdev — 子设备（如传感器、ISP）

```c
struct v4l2_subdev {
    const struct v4l2_subdev_ops *ops;
    struct v4l2_ctrl_handler *ctrl_handler;
    /* ... */
};
```

### 2.3 video_device — 对应 /dev/videoN

```c
struct video_device {
    const struct v4l2_ioctl_ops *ioctl_ops;
    const struct v4l2_file_operations *fops;
    struct vb2_queue *queue;         // videobuf2 队列
    /* ... */
};
```

### 2.4 vb2_queue — videobuf2 缓冲区队列

```c
struct vb2_queue {
    unsigned int type;               // V4L2_BUF_TYPE_VIDEO_CAPTURE
    unsigned int io_modes;           // VB2_MMAP | VB2_DMABUF
    const struct vb2_ops *ops;
    const struct vb2_mem_ops *mem_ops; // vb2_vmalloc_memops / vb2_dma_contig_memops
    /* ... */
};
```

## 3. 驱动初始化流程

```
probe()
  ├── v4l2_device_register()           // 注册 V4L2 设备
  ├── video_device_alloc()             // 分配 video_device
  ├── vb2_queue_init()                 // 初始化 videobuf2 队列
  ├── v4l2_ctrl_handler_init()         // 初始化控制句柄（曝光/增益等）
  └── video_register_device()          // 注册 /dev/videoN
```

## 4. 缓冲区管理

### 4.1 MMAP 模式（用户态 mmap 物理内存）

```c
// 用户态
VIDIOC_REQBUFS    -> 申请 N 个缓冲区
VIDIOC_QUERYBUF   -> 查询缓冲区偏移
mmap()            -> 映射到用户地址空间
VIDIOC_QBUF       -> 入队缓冲区
VIDIOC_STREAMON   -> 开始采集
VIDIOC_DQBUF      -> 出队（取帧）
```

## 5. 用户态访问接口

### 最小可用采集程序框架

```c
int fd = open("/dev/video0", O_RDWR);

// 设置格式
struct v4l2_format fmt = {0};
fmt.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
fmt.fmt.pix.width  = 3840;
fmt.fmt.pix.height = 2160;
fmt.fmt.pix.pixelformat = V4L2_PIX_FMT_NV12;  // IMX415 常用格式
ioctl(fd, VIDIOC_S_FMT, &fmt);

// 申请缓冲区
struct v4l2_requestbuffers req = {0};
req.count  = 4;
req.type   = V4L2_BUF_TYPE_VIDEO_CAPTURE;
req.memory = V4L2_MEMORY_MMAP;
ioctl(fd, VIDIOC_REQBUFS, &req);

// ... mmap, QBUF, STREAMON, DQBUF 循环 ...
```

### 6.2 常见问题

| 问题 | 可能原因 | 解决方向 |
|------|----------|----------|
| `/dev/video0` 不存在 | 驱动未加载或设备树未启用 | `dmesg | grep imx415` |
| 图像花屏 | 时序/格式不匹配 | 检查 `V4L2_PIX_FMT` 设置 |
| 帧率低 | DMA 带宽不足 | 使用 DMABUF 零拷贝 |

## 7. 调试技巧

```bash
# 查看已注册的 V4L2 设备
v4l2-ctl --list-devices

# 查询设备支持的格式
v4l2-ctl -d /dev/video0 --list-formats-ext

# 抓取单帧测试
v4l2-ctl -d /dev/video0 --stream-mmap --stream-count=1 -o frame.raw

# 查看内核日志
dmesg | grep -i "v4l2\|csi\|imx415"
```
