```

  │ signal: updateDht11Data(QString humi, QString temp)

              │
              ├── lblTemp->setText()

  │ signal: imageReady(QImage)
  └──────→ Monitor::showImage(QImage)
              └── videoLabel->setPixmap()

SpeechRecognition
  │ signal: RecordFinished()
  └──────→ MainWindow::on_handleRecord()  →  百度ASR  →  LLM  →  TTS

MainWindow ──signal──→ MusicPlayer::on_handleCommand(int)
```

## 四、摄像头相关代码详解（★ 重点）

### 4.1 现有两套采集线程对比

| | CaptureThread（旧，Monitor 用） | CameraThread（新，camera_qt 独立测试） |
|---|---|---|
| **文件** | `Monitor/capture_thread.h/cpp` | `camera_qt/camera_thread.h/cpp` |
| **设备节点** | `/dev/video0` | `/dev/video11` |
| **V4L2 类型** | `VIDEO_CAPTURE`（单平面） | `VIDEO_CAPTURE_MPLANE`（多平面） |
| **像素格式** | RGB565 | NV12 |
| **分辨率** | 1024×600 | 640×480 |
| **buffer 数** | 3 | 4 |
| **格式转换** | 不需要（直接 `Format_RGB16`） | 需要 `nv12ToQImage()` |
| **输出信号** | `imageReady(QImage)` | `frameReady(QImage)` |
| **限速** | 无（全速跑） | `msleep(33)` 约 30fps |
| **当前状态** | 被 `DISABLE_HARDWARE` 宏禁用 | 独立可运行 |

### 4.2 CaptureThread 完整接口（旧 — Monitor 模块用的）

```cpp
// Monitor/capture_thread.h
class CaptureThread : public QThread {
    Q_OBJECT
signals:
    void imageReady(QImage);       // 每帧发一次，QImage 格式 RGB16
    void sendImage(QImage);        // 备用（未使用）

public:
    void startThread();            // startFlag=true, 启动线程
    void stopThread();             // startFlag=false, 退出线程

public slots:
    void setThreadStart(bool);     // 外部控制开/关
    void setLocalDisplay(bool);    // 控制是否 emit imageReady
};
```

V4L2 参数（硬编码在 cpp 中）：
```cpp
#define VIDEO_DEV          "/dev/video0"
#define VIDEO_BUFFER_COUNT  3
// 格式：V4L2_BUF_TYPE_VIDEO_CAPTURE, RGB565, 1024×600
```

### 4.3 CameraThread 完整接口（新 — ISP MPLANE）

```cpp
// camera_qt/camera_thread.h
class CameraThread : public QThread {
    Q_OBJECT
signals:
    void frameReady(QImage img);   // 每帧发一次，QImage 格式 RGB888（NV12转换后）

public:
    explicit CameraThread(QObject *parent = nullptr);
    void stop();                   // 设 running=false，线程退出

protected:
    void run() override;           // 线程主函数（初始化→循环取帧→cleanup）

private:
    bool initDevice();             // 初始化 V4L2（open→S_FMT→REQBUFS→mmap→QBUF→STREAMON）
    void closeDevice();            // STREAMOFF→munmap→close
    QImage nv12ToQImage(void *data, int w, int h);  // NV12→RGB888 格式转换

    int fd;                        // 设备文件描述符
    CamBuffer buffers[BUF_COUNT];  // mmap 后的 buffer 数组
    volatile bool running;         // 线程运行标志
};
```

V4L2 参数（camera_thread.h 宏定义）：
```cpp
#define DEVICE     "/dev/video11"
#define WIDTH       640
#define HEIGHT      480
#define BUF_COUNT   4
// 格式：V4L2_BUF_TYPE_VIDEO_CAPTURE_MPLANE, NV12, 640×480
```

## 五、把摄像头画面接入主界面的方案

**目标**：主界面的 `cameraLabel`（中栏）实时显示摄像头画面，而不是只在 Monitor 独立窗口里看。

### 方案 A：复用现有 CaptureThread（改动最少）

**步骤 1**：在 `Monitor` 中暴露 captureThread 指针

```cpp
// Monitor/monitor.h — public 区域添加：
public:
    CaptureThread* getCaptureThread() { return captureThread; }
```

**步骤 2**：在 MainWindow 中连接信号

```cpp
// mainwindow.cpp — setupConnections() 中添加：
connect(monitor->getCaptureThread(), &CaptureThread::imageReady,
        this, [this](QImage img) {
    cameraLabel->setPixmap(
        QPixmap::fromImage(img).scaled(
            cameraLabel->size(), Qt::KeepAspectRatio, Qt::FastTransformation)
    );
});
```

**步骤 3**：去掉 `DISABLE_HARDWARE` 宏，启动时开始采集

```cpp
// VehicleTerminal.pro 中删除：
DEFINES += DISABLE_HARDWARE

// mainwindow.cpp 构造函数末尾添加：
monitor->myStart();
```

### 方案 B：换成新的 CameraThread（用 ISP /dev/video11 + NV12）

**步骤 1**：把 `camera_qt/camera_thread.h/cpp` 复制到 `Monitor/` 或项目根目录

**步骤 2**：在 `.pro` 文件中添加新文件

```pro
SOURCES += camera_thread.cpp
HEADERS += camera_thread.h
```

**步骤 3**：在 MainWindow 中直接使用

```cpp
// mainwindow.h — 添加成员：
#include "camera_thread.h"
CameraThread *cameraThread;

// mainwindow.cpp — 构造函数中：
cameraThread = new CameraThread(this);
connect(cameraThread, &CameraThread::frameReady,
        this, [this](QImage img) {
    cameraLabel->setPixmap(
        QPixmap::fromImage(img).scaled(
            cameraLabel->size(), Qt::KeepAspectRatio, Qt::FastTransformation)
    );
});
cameraThread->start();

// 析构函数中：
cameraThread->stop();
cameraThread->wait();
```

### 两个方案对比

| | 方案A（复用 CaptureThread） | 方案B（用新 CameraThread） |
|---|---|---|
| 改动量 | 3 处（暴露指针 + connect + 去宏） | 5 处（拷文件 + .pro + 新成员 + connect + 析构） |
| 设备节点 | `/dev/video0`（USB 摄像头） | `/dev/video11`（ISP MPLANE） |
| 像素格式 | RGB565（无需转换，最快） | NV12（需 CPU 转 RGB，有开销） |
| 分辨率 | 1024×600 | 640×480（可改） |
| 稳定性 | 正点原子原版代码，验证过 | 自己写的，已在 camera_qt 独立测试通过 |

### 板子上编译测试流程

```bash
# ====== 第一步：加载雷达驱动（如果要用雷达功能） ======
# RadarReader 线程会打开 /dev/fmcw_radar，驱动没加载则雷达功能不可用（不影响其他功能）
cd /home/cat/chezai/driver/fmcw_radar
sudo insmod fmcw_radar.ko
# 验证：
ls /dev/fmcw_radar   # 能看到就OK
    #后面要修改设备树
    # 先解绑 spi0.0（如果雷达接的是 spi0.0，否则换成 spi0.1）
    echo spi0.0 | sudo tee /sys/bus/spi/drivers/spidev/unbind

    # 绑定给 fmcw_radar
    echo spi0.0 | sudo tee /sys/bus/spi/drivers/fmcw_radar/bind

    # 验证
    ls /dev/fmcw_radar
# 如果不需要雷达，可以跳过这步，程序不会崩，只是雷达扫描图没数据

# ====== 第二步：编译主程序 ======
cd /home/cat/chezai/VehicleTerminal
qmake VehicleTerminal.pro    # 重新生成 Makefile（因为改了 .pro）
make clean                    # 清除旧编译产物（可选，保险起见）
make -j4                      # 4核并行编译

# ====== 第三步：运行 ======
./VehicleTerminal

# 如果摄像头画面没出来，检查：
# 1. /dev/video11 是否存在
ls /dev/video11
# 2. 权限是否够
sudo chmod 666 /dev/video11
# 3. 是否被其他进程占用
sudo fuser /dev/video11
```

**注意事项：**
- 摄像头用的是 `/dev/video11`（ISP MPLANE），不是 USB 摄像头
- `DISABLE_HARDWARE` 只禁用旧的 `CaptureThread` / `Dht11` / `Ap3216c`，**不影响新的 `CameraThread`**
- 雷达驱动和摄像头互不依赖，可以只开一个
