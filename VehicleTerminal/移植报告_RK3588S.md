# IMX6ULL 车载终端系统移植到 RK3588S (鲁班猫4) 报告

## 一、移植概述

| 项目 | 原平台 | 目标平台 |
|------|--------|----------|
| **SoC** | NXP i.MX6ULL | Rockchip RK3588S |
| **架构** | ARM Cortex-A7 (32位) | ARM Cortex-A76/A55 (64位) |
| **宏定义** | `__arm__` | `__aarch64__` |

---

## 二、主要修改内容

### 2.1 架构宏定义修改

**问题**：原代码使用 `#if __arm__` 判断ARM平台，但ARM64平台定义的是 `__aarch64__`。

**修改文件**：
- `dht11.cpp`
- `capture_thread.cpp`
- `gps.c`
- `ap3216.cpp`

**修改方式**：
```c
// 修改前
#if __arm__

// 修改后
#if defined(__arm__) || defined(__aarch64__)
```

---

### 2.2 设备路径适配

**问题**：不同平台的串口和摄像头设备节点不同。

| 设备 | IMX6ULL | RK3588S |
|------|---------|---------|
| GPS串口 | `/dev/ttymxc2` | `/dev/ttyS3` |
| 摄像头 | `/dev/video1` | `/dev/video0` |

**修改文件**：`gps.c`、`capture_thread.cpp`

```c
// gps.c
#if defined(__aarch64__)  // RK3588S (ARM64)
const char* gps_device = "/dev/ttyS3";
#else  // IMX6ULL
const char* gps_device = "/dev/ttymxc2";
#endif

// capture_thread.cpp
#if defined(__aarch64__)
#define VIDEO_DEV "/dev/video0"
#else
#define VIDEO_DEV "/dev/video1"
#endif
```

---

### 2.3 添加 DISABLE_HARDWARE 宏支持

**目的**：在没有硬件驱动的情况下运行Qt界面进行测试。

**修改文件**：`VehicleTerminal.pro`

```pro
DEFINES += DISABLE_HARDWARE
```

**受影响的功能**：
| 模块 | DISABLE_HARDWARE模式行为 |
|------|-------------------------|
| DHT11温湿度 | 使用模拟数据 (28.0°C / 25.0%) |
| GPS | 使用默认坐标 (113.38, 34.35) |
| AP3216传感器 | 返回0 |
| 摄像头 | 不启动采集线程 |
| 语音控制 | 禁用 |

---

### 2.4 头文件静态变量问题修复

**问题**：`gps.h` 中使用 `static` 变量导致多重定义链接错误。

**修改前** (`gps.h`)：
```c
static char dataBuf[100];
static int gps_fd = -1;
static int gps_valid = 0;
```

**修改后**：
```c
// gps.h - extern声明
extern char dataBuf[100];
extern int gps_fd;
extern int gps_valid;
extern const char* gps_device;

// gps.c - 实际定义
char dataBuf[100];
int gps_fd = -1;
int gps_valid = 0;
#if defined(__aarch64__)
const char* gps_device = "/dev/ttyS3";
#else
const char* gps_device = "/dev/ttymxc2";
#endif
```

---

### 2.5 Qt信号槽连接问题修复

**问题**：`connectSlotsByName` 警告提示信号不匹配。

**原因**：Qt的自动连接机制尝试连接不存在的信号。

**解决**：这些是警告而非错误，不影响功能。手动 `connect()` 已正确建立连接。

---

### 2.6 事件过滤器崩溃修复

**问题**：`MainWindow::eventFilter` 对所有事件类型进行鼠标事件类型转换导致崩溃。

**修改文件**：`mainwindow.cpp`

```cpp
// 修改前：对所有事件做类型转换
QMouseEvent* event = static_cast<QMouseEvent*>(ev);

// 修改后：先检查事件类型
if (ev->type() != QEvent::MouseButtonPress && 
    ev->type() != QEvent::MouseButtonRelease) {
    return QWidget::eventFilter(watched, ev);
}
QMouseEvent* event = static_cast<QMouseEvent*>(ev);
```

---

### 2.7 变量初始化修复

**问题**：`GifSum` 未初始化可能导致随机值。

**修改文件**：`settingwindow.h`

```cpp
// 修改前
int GifSum;

// 修改后
int GifSum = 0;
```

---

### 2.8 pthread读写锁初始化

**问题**：DISABLE_HARDWARE模式下 `pthread_rwlock_t` 未初始化。

**修改文件**：`gps.c`

```c
int gps_init()
{
    // 始终初始化读写锁（不论是否启用硬件）
    pthread_rwlock_init(&gps_rwlock, NULL);
    
#if (defined(__arm__) || defined(__aarch64__)) && !defined(DISABLE_HARDWARE)
    // 硬件初始化代码...
#else
    gps_valid = 1;  // 使用默认坐标
#endif
}
```

---

### 2.9 添加网络模块

**问题**：链接错误缺少 `QNetworkAccessManager` 等类。

**修改文件**：`VehicleTerminal.pro`

```pro
QT += network
```

## 三、音乐功能问题分析

### 3.1 问题现象
```
defaultServiceProvider::requestService(): no service found for - "org.qt-project.qt.mediaplayer"
```

### 3.2 原因分析

**原因1：缺少Qt Multimedia后端**

RK3588S系统需要安装GStreamer插件：
```bash
sudo apt install gstreamer1.0-plugins-base gstreamer1.0-plugins-good \
    gstreamer1.0-plugins-bad gstreamer1.0-plugins-ugly \
    libqt5multimedia5-plugins
```

**原因2：音乐文件夹位置**

程序在 `可执行文件目录/myMusic` 查找音乐，需要复制文件夹：
```bash
# 假设程序在 ~/chezai/VehicleTerminal 目录
cp -r VehicleTerminal/Music/myMusic ~/chezai/VehicleTerminal/
```

## 四、编译与运行步骤

### 4.1 依赖安装 (鲁班猫4)

```bash
sudo apt update
sudo apt install qt5-default qtmultimedia5-dev libqt5webkit5-dev \
    gstreamer1.0-plugins-base gstreamer1.0-plugins-good \
    libqt5multimedia5-plugins
```

### 4.2 编译
```bash
cd VehicleTerminal
qmake
make -j4
```

```bash
# 复制音乐和GIF文件夹
cp -r Music/myMusic ./
mkdir -p MyGif
# 运行程序
./VehicleTerminal
```
